# Idea 详情页优化方案

> 目标：构建逻辑清晰、说服力强、能驱动用户订阅的详情页体验

---

## 一、当前问题诊断

### 1.1 逻辑断层

| 当前模块顺序 | 问题 |
|-------------|------|
| 1. Hero (标题+评分) | ✅ 合理 |
| 2. 趋势图表 | ⚠️ 过早出现，用户还未理解这是什么机会 |
| 3. 用户痛点 | ✅ 合理 |
| 4. 竞品分析 | ✅ 合理（新功能） |
| 5. Why Now | ⚠️ 位置尴尬，应在结论前作为支撑 |
| 6. 需求来源 | ⚠️ 与 Why Now 内容重叠 |
| 7. Full Breakdown | ⚠️ 过长，信息密度不均 |
| 8. 侧栏：Quick Facts | ❌ 元数据对用户决策无帮助 |
| 9. 侧栏：变现/执行/关键词 | ⚠️ 与主栏内容割裂 |

### 1.2 主栏 vs 侧栏定位模糊

- **侧栏当前内容**：阅读时间、难度、时间线、发布日期、变现方式、执行步骤、关键词
- **问题**：
  - Quick Facts 是"文章元数据"而非"机会评估数据"
  - 变现和执行步骤被挤在侧栏，展示空间有限
  - 用户无法快速判断"这个机会是否适合我"

### 1.3 订阅驱动力不足

- 缺乏"钩子"让用户感知持续价值
- 没有展示内容深度和专业度的证明
- CTA 区域过于简单（仅两个按钮）

## 二、新信息架构设计

### 2.1 核心原则

1. **倒金字塔结构**：结论先行 → 证据支撑 → 行动指引
2. **用户决策路径**：这是什么 → 为什么值得做 → 怎么做 → 风险是什么
3. **主栏重叙事，侧栏重决策**：主栏讲故事，侧栏给判断依据

### 2.2 新模块顺序（主栏）


## 三、关键改进点详解

### 3.1 Hero 区域重构

**当前**：标题 + 描述 + 三个 summary 卡片（verdict/confidence/bestWedge）

**新设计**：
- **标题区**：保持现有
- **机会评分卡**：四维度可视化雷达图或进度条
  - 需求强度（基于 trend + pain signal）
  - 竞争空间（基于 competitor analysis）
  - 进入门槛（基于技术难度 + 资源要求）
  - 变现潜力（基于 pricing 空间 + market size）
- **核心结论**：保留 verdict，但强化视觉（大标签 + emoji）

### 3.2 证据层整合

**当前**：趋势图表单独一节，需求来源一节，Why Now 一节

**新设计**：
- 合并为"需求证据"一节
- 顶部展示 3 个信号卡片（搜索趋势、社区热度、竞品缺口）
- 下方展开趋势图表（可选）
- 移除单独的 Why Now（内容并入痛点描述）

### 3.3 MVP 路线图视觉化

**当前**：侧栏简单的 1-2-3 步骤列表

**新设计**：
- 移入主栏，改为**时间轴形式**
- 每个阶段包含：时间、里程碑、关键动作、验证标准
- 视觉：横向或纵向时间轴，带进度状态

### 3.4 变现层强化

**当前**：侧栏简单列表

**新设计**：
- 移入主栏，展示：
  - 推荐定价模式（基于竞品分析）
  - 收入估算计算器（简单交互：输入用户数 → 估算月收入）
  - 竞品定价对比表

### 3.5 新增风险层

**当前**：无

**新设计**：
- 新增"风险与注意事项"一节
- 包含：技术风险、市场风险、执行风险
- 每个风险配缓解策略
- 明确标注"不适合此机会的人群"

### 3.6 Full Breakdown 折叠化

**当前**：默认展开，过长

**新设计**：
- 默认折叠，标题为"深度技术/市场拆解"
- 点击展开，保持现有内容结构
- 减少首屏信息过载

### 3.7 CTA 区域强化

**当前**：简单两个按钮

**新设计**：
- 价值主张重申（"每周 3 个验证机会"）
- 内嵌邮件订阅表单（无需跳转）
- 社交证明（订阅者数量、 testimonials）
- 分享按钮（Twitter/X、LinkedIn、复制链接）

## 四、数据需求

为支持新设计，需要 pipeline 生成以下新数据字段：

| 字段 | 类型 | 用途 |
|------|------|------|
| `opportunityScore` | Object {demand, competition, barrier, monetization} | 机会评分卡 |
| `marketSize` | Object {tam, sam, som} | 市场规模估算 |
| `risks` | Array<{description, mitigation}> | 风险层 |
| `mvpRoadmap` | Array<{phase, duration, milestones, validation}> | MVP 时间轴 |
| `revenueEstimate` | Object {model, unitPrice, assumptions} | 收入估算 |
| `notFor` | Array<string> | 不适合人群 |

## 五、实施优先级

### Phase 1：结构调整（高优先级）
1. 重构模块顺序（主栏）
2. 重新设计侧栏为决策仪表盘
3. 移除/替换 Quick Facts
4. Full Breakdown 折叠化

### Phase 2：内容增强（中优先级）
1. 新增风险层
2. MVP 路线图视觉化
3. 变现层强化
4. CTA 区域强化

### Phase 3：数据升级（低优先级）
1. 接入新的 pipeline 数据字段
2. 机会评分卡可视化
3. 收入估算计算器

## 六、预期效果

- **信息架构**：逻辑清晰，符合用户决策路径
- **转化提升**：侧栏决策仪表盘 + 强化 CTA 提升订阅率
- **专业感知**：风险层、数据可视化增强可信度
- **阅读体验**：折叠化减少信息过载，首屏聚焦核心价值
